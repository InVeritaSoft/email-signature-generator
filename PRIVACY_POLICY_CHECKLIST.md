# Privacy Policy Checklist for Chrome Web Store Submission

## ✅ Before Publishing

### 1. Update Contact Information
In `privacy-policy.html`, replace:
- `[Your contact email]` → Your actual contact email
- `[Your GitHub repository URL]` → Your actual GitHub repo URL (if applicable)

### 2. Update Last Updated Date
- Change "November 2024" to the current date when you publish

### 3. Host on GitHub Pages
- Follow instructions in `GITHUB_PAGES_SETUP.md`
- Get your public URL (e.g., `https://username.github.io/repo/privacy-policy.html`)
- Test the URL in an incognito browser to ensure it's publicly accessible

### 4. For Chrome Web Store Submission
- Go to Chrome Web Store Developer Dashboard
- Navigate to your extension listing
- Go to **Privacy practices** section
- Enter your GitHub Pages URL in the **Privacy Policy URL** field
- Example: `https://yourusername.github.io/email-signature-generator/privacy-policy.html`

## Quick Test

Before submitting, verify:
1. ✅ Privacy policy URL is publicly accessible
2. ✅ URL uses HTTPS (GitHub Pages provides this automatically)
3. ✅ Page loads correctly on mobile devices
4. ✅ All contact information is filled in
5. ✅ Last updated date is current
